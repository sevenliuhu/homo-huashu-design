## 核心理念

### 0. 事实验证先于假设（规格升级版）

**任何涉及具体品牌/产品/事件的存在性、发布状态、规格，第一步必须 WebSearch 验证。**

**中式特殊检查项**：
- 企业品牌名可能有同名竞争（如「华为」要确认是通信华为还是其他）
- 电商品牌可能有多个子品牌线（如「完美日记」下的小奥汀、完子心选）
- 促销活动时间必须验证是否为真实大促（618/双11）

### 1. 中文设计的独特美学

中文设计的认知底层与西方不同：

| 维度 | 西方 | 中文 |
|------|------|------|
| 留白 | 功能导向 | 诗意导向（虚实相生） |
| 色彩 | 品牌色为主 | 红色=庆典/喜庆/国潮 |
| 字体 | 字距统一 | 字面率/中宫/重心 |
| 信息密度 | 宽松 | 可接受更高密度 |
| 用户习惯 | 左到右 | 同，但竖向排版有传统基础 |
| 数字感知 | 千分位 | 万分位（如 10,000 写「1万」） |

### 2. 中文版 vs 国际版执行指南

| 要素 | 国际版 | 中文企业版 |
|------|--------|-----------|
| 默认字体 | Inter / SF Pro / Roboto | 思源黑体 / 阿里巴巴普惠体 / 汉仪 |
| 数字 | 1,234,567.89 | 123.46万 / 1.23亿 |
| 日期 | 2026-05-22 | 2026年5月22日 |
| 电话 | +86 10 8888 8888 | 400-888-8888（服务热线） |
| 地址 | Street, City, ZIP | 省市区街道门牌 |
| 价格 | $29.99 | ¥29.90 / 29.9元 |
| 节庆 | Christmas, Black Friday | 春节 / 618 / 双11 / 中秋 |
| 社交 | Twitter, Instagram, LinkedIn | 微信 / 抖音 / 小红书 / B站 |

### 3. 反 AI slop（中文版特化）

除了原始 slop 清单，中文场景额外规避：

| 元素 | 为什么是 slop |
|------|--------------|
| 书法字乱飞 | 以为加了书法就显「中国风」，实际廉价 |
| 大红配大黄 | 以为「喜庆」= 国旗配色，实际俗 |
| 龙/祥云滥用 | 以为有龙=国潮，实际过时 |
| 圆角大红按钮 | 淘宝模板风，无品牌辨识度 |
| 毛笔触感 SVG 装饰 | 画虎不成反类犬的典型 |
| 全是「第X届」「XX之夜」 | 发布会模板感 |
| 伪黑体（微软雅黑）做 display | 辨识度低，无设计感 |

**正能量方向**：
- ✅ 宋体/仿宋的现代演绎（字魂・标小智龙明、方正悠宋）
- ✅ 东方色彩体系（胭脂/黛蓝/月白/松花/竹青）
- ✅ 山水意境的抽象化（不是直接画山水，是用山形曲线构版面）
- ✅ 器物的现代转译（琮/鼎/瓷器的几何化）
- ✅ 汉字本身的图形化潜力（字体的构造美）
- ✅ 留白中的气息（与西方网格留白的本质差异）

### 4. 品牌资产协议 - 中文版增强

在原始 5 步流程基础上，中文品牌增加以下步骤：

**Step 0 · 品牌域名核实**
中文品牌常见情况：
- 同名不同赛道（如「小米」有手机/家电/汽车/生态链）
- 英文名+中文名不统一（如 MEIZU=魅族，但不是所有拼音名=品牌名）
- 企业品牌VS产品品牌（如阿里巴巴集团下有淘宝/天猫/1688/盒马等）

**Step 1 · 中文品牌资产清单**

```
关于 <品牌>，你手上有以下哪些资料？我按优先级列：

1. Logo（SVG / 高清 PNG / AI / EPS）—— 必须！需确认中文版+英文版
2. 产品图 / 商品主图（800×800px+ / 白底图 / 场景图）
3. 品牌色体系（主色+辅色+辅助色+渐变规则）
4. 字体系列（中文+英文，display+body）
5. 品牌规范文档（brand guideline PDF / Figma）
6. 电商渠道链接（天猫/京东/抖音旗舰店 URL）
7. 社交媒体账号（微信公众号/小红书/抖音/B站）
8. 近期营销活动/海报（了解当前视觉风格）
9. 竞品/对标品牌参考
10. 产品定价区间与用户画像
```

**Step 2 · 中文品牌搜索路径**

| 资产 | 搜索路径 |
|------|---------|
| Logo | 官网(nav/header内SVG) · 招聘页 · 新闻稿 · 淘宝/京东旗舰店header |
| 产品图 | 天猫旗舰店商品页 · 京东自营 · 抖音商品橱窗 · 官网产品页 |
| 用户评价图 | 小红书搜索 · 什么值得买 · 大众点评 |
| 促销 Banner | 淘宝首页搜索 · 京东大促页 · 抖音直播间背景 |
| 品牌色 | 官网CSS · 小程序代码 · 天猫店铺装修CSS |
| 字体 | 官网 `<link>` · 微信文章用字 · 小程序font-family |

**Step 3 · 下载特殊处理**

```bash
# 中文品牌 Logo 下载（网站可能使用中文域名）
curl -A "Mozilla/5.0" -L "https://www.<品牌>.com" -o assets/<品牌>-brand/homepage.html
# 从 HTML 提取 SVG logo
grep -oP '<svg[^>]*>.*?</svg>' assets/<品牌>-brand/homepage.html > assets/<品牌>-brand/logo.svg

# 加宽搜索：如果主站找不到，搜索天猫/京东店铺
curl -A "Mozilla/5.0" -L "https://detail.tmall.com/item.htm?id=<商品ID>" -o assets/<品牌>-brand/tmall-product.html
```

**Step 4 · 中文色值提取**

```bash
# 提取中文品牌色（不仅从CSS，也从天猫店铺装修取）
grep -hoE '(#[0-9A-Fa-f]{6})|(#[0-9A-Fa-f]{3})|(rgb\([^)]+\)|rgba\([^)]+\))' assets/<品牌>-brand/* \
  | sort | uniq -c | sort -rn | head -20

# 中文品牌也检查小程序/App内的色值
```

### 5. 中文展示（Tweaks参数库）

比国际版增加的 Tweaks 参数：

| 参数 | 值范围 | 说明 |
|------|--------|------|
| 字体族 | 思源黑体/阿里巴巴普惠体/汉仪旗黑/方正悠宋 | 中文字体选择 |
| 排版风格 | 现代/国潮/极简/人文 | 整体排版气质 |
| 数字格式 | 万/亿/1234.56万 | 大数字表示法 |
| 节日模式 | 无/中秋/春节/618/双11 | 自动切换配色元素 |
| 信息密度 | 疏/中/密 | 适应不同场景 |
| 中英文比例 | 纯中文/中英混排/纯英文 | 双语场景 |
| 地域感知 | 国内/海外 | 适配不同市场 |

### 新增 5 种中文专有设计语汇（HOMO 增强）

#### 21. 东方国潮 - 新中式品牌美学
**哲学**：传统不是包袱，是 IP 金矿
**核心特征**：
- 中国传统色彩体系（胭脂 #C03C3C · 黛蓝 #425066 · 月白 #E6EAF0 · 松花 #A5B37A）
- 汉字设计作为核心视觉语言（字型解构/笔画重组）
- 传统纹样的现代化演绎（回纹/云纹/冰裂纹的几何化）
- 中式留白哲学（不是西方"空"，是东方"气"）
- 红色 + 墨色 + 金 / 银的点缀

**提示词DNA**：
```
Chinese trendy / 国潮 aesthetic:
- Traditional Chinese color palette (rouge #C03C3C, ink blue #425066)
- Chinese typography as primary visual element
- Modern interpretation of traditional patterns
- Red + ink black as base palette
- Gold/silver accent for luxury feel
- Square format compositions inspired by seal carving
- Negative space with poetic intent (Qi/气 principle)
- Print-making aesthetic (woodblock/letterpress feel)
- Chinese calligraphy strokes as graphic elements
```

**适用场景**：国货美妆/白酒/茶叶/文创/食品/汉服
**代表作**：花西子、观夏、霸王茶姬

---

#### 22. 互联网大厂风 - 增长黑客 UI
**哲学**：数据驱动的增长，干净但有转化力
**核心特征**：
- 大字号（标题 48-96px）
- 毛玻璃/弥散光效
- 清晰的 CTA 层级（主按钮/次按钮/文字链）
- 数据仪表盘的美学
- 渐变的克制使用（品牌渐变，非紫蓝渐变）
- 引导性的动效（hover/微交互动画）

**提示词DNA**：
```
Chinese big-tech UI style:
- Large typography (headlines 48-96px)
- Glassmorphism / frosted glass effects
- Clean CTA hierarchy
- Dashboard-inspired layout grids
- Brand-consistent gradients only
- Conversion-oriented white space
- Micro-interactions on all interactive elements
- Official account / mini-program visual language
```

**适用场景**：SaaS产品/App推广页/小程序/工具类产品
**代表作**：微信小程序官方设计、飞书设计规范

---

#### 23. 小红书/抖音 - 社交平台内容美学
**哲学**：信息流中的停留 = 设计的目标
**核心特征**：
- 竖版视频/图文混合布局（9:16 竖屏优先）
- 高对比度 + 柔和光晕
- 封面图承载 60% 的点击率
- 文字极少但信息明确（标题 ≤ 15 字）
- 无缝适配信息流环境
- 生活化而非精致过度（Too perfect = 不真实）

**提示词DNA**：
```
Social platform content aesthetic (Xiaohongshu / Douyin style):
- Vertical format (9:16) with mobile-first layout
- High contrast with soft glow effects
- Cover image as primary attention grabber
- Minimal text (headlines ≤ 15 characters)
- Native feed integration design
- Lifestyle feel, not overly polished
- Warm tone, approachable, genuine
- Thumb-stopping color combinations
- User-generated-content visual language
```

**适用场景**：社交媒体运营图/小红书笔记封面/抖音视频封面/种草图文
**代表作**：小红书头部博主视觉、抖音热门封面

---

#### 24. 中式电商直击 - 转化率驱动设计
**哲学**：每一帧都在卖，但不是硬卖
**核心特征**：
- 价格信息的视觉武器化（原价划掉/到手价放大）
- 信任信号的嵌入（销量/好评率/官方认证标签）
- 场景化商品展示（使用场景而非单纯产品白底）
- 促销紧迫感的设计（倒计时/限量进度条）
- 信息密度可接受（中国人习惯更多内容）
- 移动端竖版优先（90% 流量来自手机）

**提示词DNA**：
```
Chinese e-commerce conversion design:
- Price as hero element (original price strikethrough, price emphasis)
- Trust signals integrated (sales count, rating, verified badges)
- Contextual product display (usage scenarios, not just white background)
- Urgency design (countdown timers, limited stock bars)
- Higher information density
- Mobile-first vertical layout
- Multi-layered value proposition
- Social proof widgets integrated into design
- Bundle/套餐 visual representation
```

**适用场景**：天猫详情页/抖音直播预告/电商广告图/促销 Banner
**代表作**：李佳琦直播间视觉、天猫双11主视觉

---

#### 25. 中式发布会 - 科技感+人文感
**哲学**：科技企业的"舞台感"设计
**核心特征**：
- 深色背景为主（深灰/深蓝/黑）
- 超大文本作为视觉焦点（40pt+ 投影安全字号）
- 产品渲染图的分层展示
- 数据可视化作为叙事结构
- 灯光/辉光效果（舞台感）
- 克制的中文排版（中英文混排规范）

**提示词DNA**：
```
Chinese tech keynote design:
- Dark background (charcoal, deep blue, pure black)
- Oversized typography (40pt+ safe for projection)
- Layered product renderings
- Data visualization as narrative structure
- Stage lighting effects (glow, spotlight)
- Controlled Chinese-English bilingual typesetting
- Keynote-projection safe (high contrast, big elements)
- Brand reveal moments (name in big type)
- Transition-friendly composition
```

**适用场景**：新品发布会/年度演讲/品牌升级发布会
**代表作**：华为/小米发布会 Keynote、微信公开课 Pro

## 电商场景模板

### E1. 天猫/京东详情页

**规格**：
- 宽度 790px（天猫标准）/ 990px（京东标准）
- 高度不限，长页面滚动
- 首屏高度 ≈ 视窗

**核心模块**：
1. 首屏主图区（白底商品图 / 模特图 / 使用场景图）
2. 核心卖点陈列（3大优势图标化）
3. 产品参数对比表格
4. 细节特写（材质/做工/功能特写）
5. 使用场景展示（3-5个使用场景）
6. 用户评价/口碑展示
7. 品牌背书/资质证书
8. 购买引导区（规格选择+立即购买）

**推荐风格**：21 东方国潮 / 24 中式电商 / 01 Pentagram（国际品牌）

### E2. 抖音/快手直播预告

**规格**：
- 竖版 1080×1920px
- 文字大，信息单一（3秒看完）

**核心设计**：
- 直播时间（"今晚8点" 大字号）
- 核心卖点（1-2个最吸引人的点）
- 价格信息 / 优惠力度
- 主播头像（可选）
- 动作引导（"点击预约" / "来我直播间"）
- 品牌 Logo 水印

**推荐风格**：23 社交平台 / 24 中式电商

### E3. 电商大促 Banner

**规格**：
- 750×420px（天猫 Banner）/ 多种尺寸适配
- 文字不超过 15 字

**核心要素**：
- 促销力度（"满199减30" / "第二件半价"）
- 大促 IP（618 / 双11 / 年货节 / 女神节等）
- 品牌 + 产品主图
- CTA 按钮（"立即抢购"）
- 紧迫感元素（限时 / 限量 / 专区）

**推荐风格**：24 中式电商 / 21 东方国潮 / 12 快乐极简

### E4. 电商品牌海报

**规格**：
- 1080×1440px（3:4 小红书/朋友圈）
- 1200×628px（微信分享缩略图）

**设计要点**：
- 品牌调性第一，促销第二
- 产品图品质感（光影/细节）
- 品牌 Slogan + Logo 明确
- 配色与品牌一致
- 避免过度促销感（高级感优先）

**推荐风格**：21 东方国潮 / 11 Build / 17 Takram / 18 Kenya Hara

### E5. 电商详情分析页

**规格**：PDF / 长图 / HTML
**包含**：销售数据可视化、用户画像分析、流量来源、转化漏斗

**推荐风格**：04 Fathom / 10 Müller-Brockmann / 02 Stamen

详情见 `references/ecommerce.md`

## 5维专家评审（中文版增强）

保留原始 5 维度（哲学一致性/视觉层级/细节执行/功能性/创新性），增加中文版评审特化：

### 附加评审维度：文化适配度

| 分数 | 标准 |
|------|------|
| 9-10 | 完美适配中国文化语境，既有辨识度又不落俗套 |
| 7-8 | 整体方向正确，个别文化元素使用贴切 |
| 5-6 | 没有明显的文化错误，但统一性不够 |
| 3-4 | 存在文化元素的生搬硬套 |
| 1-2 | 明显的不适配（如西方排版直接套中文） |

**评审要点**：
- 字号是否考虑了中文的字面率和识别距离？（中文 14-16px body / 32px+ 标题）
- 行高是否考虑了中文字形？（中文 1.5-1.8 倍行高明显不同于英文 1.2-1.5）
- 有无文化元素的误用？（特定颜色在中/日/韩的语义差异）
- 数字表示是否符合中文习惯？（万/亿 vs千分位）

### 电商专有的评审维度：转化力

| 分数 | 标准 |
|------|------|
| 9-10 | 每个设计元素都服务于销售转化，无冗余 |
| 7-8 | CTA清晰，信息传递高效 |
| 5-6 | 基本具备电商要素，但缺乏紧迫感 |
| 3-4 | CTA不明确，信息层级混乱 |
| 1-2 | 完全不适合销售场景 |

**评审要点**：
- 价格信息是否突出识别？
- 优惠信息是否在一秒内被看到？
- CTA按钮是否足够大、颜色是否突出？
- 是否有信任信号（销量/评分/认证）？
- 信息密度是否适合移动端消费？

```
    │
    ├─ 电商场景？──── Yes ──→ 电商模板选择
    │     │                    E1~E5 模板适配
    │     │                    + 转化力评审维度
    │     │
    │     └─→ 选择设计风格 → 进入主干
    │
    ├─ 需求模糊？──── Yes ──→ 设计方向顾问（Fallback）
    │                             25 种语汇推荐 3 差异化方向

    │                             用户选方向后进入主干
    │
    └─ 需求明确 ────→ Junior Designer 工作流

                           Pass 2: 组件填充 + Variations
                           Pass 3: 细节打磨 + Tweaks
                           Pass 4: 验证 + 交付（中文版交付检查清单）
```

### 中文版交付检查清单

交付前逐项检查：

- [ ] 字体是否全部正确加载？（中文 Fallback: sans-serif，不出现微软雅黑做display）
- [ ] 数字格式是否符合中文习惯？（万/亿而不是千分位）
- [ ] 日期格式是否为中文？（2026年5月22日）
- [ ] 联系方式是否为国内格式？（400电话/手机号）
- [ ] 品牌名是否中英文一致？
- [ ] 色彩是否有文化误用？（白色在婚庆可用，在商业严肃要慎用）
- [ ] 页面宽度是否适配移动端？（375px+ 兼容 iPhone/Android）
- [ ] 是否适配微信内置浏览器？（微信 X5 Blink 内核的兼容性）
- [ ] 加载速度是否适合国内网络？（不依赖 Google Fonts 无法访问）
- [ ] 备选 CDN 是否使用了国内可用源？（cdnjs / 字节跳动 CDN）
