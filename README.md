## 中文企业品牌协议

> 在原始 5 步流程基础上的中文版增强（涉及具体品牌时强制执行）

### Step 0 · 品牌核实
同名不同赛道？英文名≠中文名？企业品牌 vs 产品品牌？先确认。

### Step 1 · 中文品牌资产清单（10项）
Logo → 产品图 → 品牌色 → 字体 → 规范文档 → 电商渠道 → 社交媒体 → 近期活动 → 竞品 → 用户画像

### Step 2 · 中文搜索路径
| 资产 | 搜索来源 |
|------|---------|
| Logo | 官网 nav · 天猫/京东店铺 header · 招聘页 |
| 产品图 | 天猫商品页 · 京东自营 · 抖音橱窗 |
| 色值 | 官网 CSS · 天猫装修 CSS · 小程序代码 |
| 字体 | 官网 `<link>` · 微信文章 · 小程序 font-family |

### Step 3 · 下载特殊处理
```bash
# 中文品牌 Logo（主站 + 电商渠道双搜索）
curl -A "Mozilla/5.0" -L "https://www.品牌.com" -o assets/<品牌>-brand/homepage.html
curl -A "Mozilla/5.0" -L "https://detail.tmall.com/item.htm?id=商品ID" -o assets/<品牌>-brand/tmall.html
```

### Step 4 · 中文色值提取
```bash
grep -hoE '(#[0-9A-Fa-f]{6})|(rgb\([^)]+\))' assets/<品牌>-brand/* | sort | uniq -c | sort -rn | head -20
```

### Step 5 · **固化 brand-spec.md**
含：Logo 路径 · 产品图路径 · 色值 · 字体 · 电商渠道 · 使用规范 · 禁区

## 工作流

### 中文企业任务流程

```
接到任务
  ├─ 涉及品牌？ → Step 0-5 中文品牌协议 → 设计方向
  ├─ 电商场景？ → 电商模板选择 + 转化力评审 → 风格选定
  ├─ 需求模糊？ → 设计方向顾问（25种语汇推3方向+并行Demo）
  └─ 需求明确 → Junior Designer 工作流
                   ↓
           Pass 1: 假设 + Placeholders（show给用户）
           Pass 2: 真实组件 + Variations（再show）
           Pass 3: 细节打磨 + Tweaks
           Pass 4: 验收 + 交付（中文交付检查清单）
```

### 中文交付检查清单

- [ ] 字体是否全部正确加载？（中文 Fallback: sans-serif，不出现微软雅黑 display）
- [ ] 数字格式使用万/亿？（非千分位）
- [ ] 日期格式使用"2026年5月22日"？
- [ ] 联系方式使用 400 电话或国内手机格式？
- [ ] 品牌名中英文一致？
- [ ] 色彩无文化误用？
- [ ] 移动端 375px 宽度兼容？
- [ ] 微信 X5 Blink 内核测试通过？
- [ ] 不依赖 Google Fonts（使用国内 CDN 或 base64）？

## 设计哲学

> **打字，回车，交付。** 不是「AI 做的还行」的水平，是看起来像大厂设计团队做的。

本增强版在保持原版核心哲学基础上（品牌资产协议、Junior Designer 工作流、反 AI slop 规则、事实验证优先），增加了中国企业特化的设计体系：

- **文化不是装饰** — 东方美学不是贴标签，是深入理解后的系统表达
- **转化不是粗俗** — 电商设计可以既有调性又有转化力
- **中文不是障碍** — 中文排版有自己独特的美学规律，需要尊重和理解
