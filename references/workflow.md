# Workflow：中文企业设计任务流

> 从接到任务到交付的中文版工作流
> 基于原始 Junior Designer 模式 + 中文企业增强

---

## 任务路由

接到任务时先判断类别：

```
类别判断
  ├─ 涉及具体中国品牌/公司名？
  │   └─→ Yes → 走中文品牌资产协议（Step 0-5）
  │
  ├─ 电商场景？（天猫/京东/抖音/拼多多/详情页/Banner）
  │   └─→ Yes → 走电商模板（E1-E5）
  │
  ├─ 涉及演讲/发布会/Keynote？
  │   └─→ Yes → 走展示模板 + 投影规范
  │
  └─ 通用设计/内容/营销？
      └─→ 走设计方向顾问或主干流程
```

## 中文品牌资产协议 5 步+

### Step 0 · 品牌核实
- 中文品牌名是否唯一？（华为通信 vs 华为...）
- 品牌子线/产品线？（如美的一共有50+事业部）
- 确认品牌/产品的最新状态

### Step 1 · 问（10 项中文清单）
```markdown
关于 <品牌>，你手上有：
1. Logo（中文版+英文版）？
2. 产品主图（白底图/场景图）？
3. 品牌色体系？
4. 字体规范？
5. Brand guideline PDF/Figma？
6. 电商渠道链接？
7. 社交媒体账号？
8. 近期设计物料？
9. 竞品参考？
10. 目标用户？
```

### Step 2 · 搜官方渠道 + 电商渠道
### Step 3 · 下载（中英双版 Logo、产品图、色值）
### Step 4 · 验证 + 提取（中文特化）
### Step 5 · 固化 brand-spec.md（中文版模板）

## 电商任务流程

### 1. 确认电商类型
- 天猫详情页 → E1
- 直播预告 → E2
- 大促 Banner → E3
- 品牌海报 → E4
- 数据分析 → E5

### 2. 确认渠道
- 天猫 vs 京东 vs 抖音 vs 拼多多 vs 私域
- 不同渠道设计规范不同（宽度/风格/尺寸）

### 3. 选择风格
根据品牌定位从 design-styles.md 选择风格

### 4. 执行
使用场景模板 + 风格DNA + 真实品牌资产

### 5. 转化力检查
交付前检查：价格/CTA/信任/紧迫感/信息层级

## 问问题指南

### 必须问的场景
- 新任务、模糊任务、无 design context
- 用户只说一句的要求

### 可以不问的场景
- 小修小补、follow-up
- 已有明确上下文的延续任务

### 一次问齐（中文版模板）
```markdown
开工前几个问题对齐：

**品牌/场景**
1. 这个设计是为哪个品牌/产品做的？
2. 目标渠道是（天猫/抖音/微信/官网/其他）？

**已有资料**
3. 有品牌规范/配色/字体/Logo吗？
4. 有产品图/UI截图可以给吗？

**风格偏好**
5. 想要什么调性？（国潮/大厂风/极简/温暖/专业）
6. 有参考竞品或参考图吗？

**输出要求**
7. 交付什么格式？（HTML/PDF/动图）
8. 几个变体？尺寸？
```

## 交付检查清单

### 核心检查
- [ ] 品牌 Logo 是否正确？（中英文双版验证）
- [ ] 品牌色值是否从资产提取而非记忆？
- [ ] 字体 Fallback 是否正确？

### 中文特化
- [ ] 数字格式为万/亿？
- [ ] 日期为中文格式？
- [ ] 联系方式为中国格式？
- [ ] 价格符号为 ¥？
- [ ] 字号适合中文阅读？

### 电商特化
- [ ] 价格信息突出？
- [ ] CTA 按钮明显？
- [ ] 信任信号可见？
- [ ] 紧迫感（如适用）？

### 技术检查
- [ ] 移动端 375px 兼容？
- [ ] 不依赖 Google Fonts？
- [ ] 微信浏览器兼容？
- [ ] 国内 CDN 可用？
- [ ] 图片已压缩？
